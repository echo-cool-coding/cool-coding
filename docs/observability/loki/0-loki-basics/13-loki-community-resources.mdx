# Loki 社区资源

## 介绍

Grafana Loki是一个开源的日志聚合系统，其活跃的社区是学习和解决问题的宝贵资源。本节将介绍如何利用Loki的官方文档、论坛、实时聊天渠道和开源协作平台，帮助你快速入门并解决实际问题。

:::tip
参与社区不仅能获取帮助，还能了解最新功能和最佳实践！
:::

## 官方资源

### 1. 文档与教程
- **官网文档**：访问 [grafana.com/docs/loki](https://grafana.com/docs/loki) 获取安装指南、配置参考和API文档。
- **Grafana Labs博客**：搜索“Loki”查看技术深度文章和版本更新。

### 2. GitHub仓库
- **主仓库**：[github.com/grafana/loki](https://github.com/grafana/loki) - 提交Issue或参与代码贡献。
- **示例项目**：查找 `loki/examples` 目录下的部署模板。

```bash
# 克隆示例仓库
git clone https://github.com/grafana/loki.git
cd loki/examples/getting-started
```

### 3. 实时交流
- **Slack**：加入 [Grafana Slack](https://slack.grafana.com/) 的 `#loki` 频道。
- **社区论坛**：在 [Grafana社区](https://community.grafana.com/) 提问或分享经验。

## 第三方资源

### 1. 视频与课程

### 2. 开源工具
- **LogCLI**：命令行工具，用于快速查询Loki日志。
  ```bash
  logcli query '{job="varlogs"}' --limit=5
  ```

## 实际案例

### 问题：如何调试日志丢失？
1. **社区提问**：在Slack描述问题场景。
2. **解决方案**：根据建议检查Promtail配置：
   ```yaml
   scrape_configs:
     - job_name: system
       static_configs:
         - targets: [localhost]
           labels:
             job: varlogs
             __path__: /var/log/*.log
   ```
