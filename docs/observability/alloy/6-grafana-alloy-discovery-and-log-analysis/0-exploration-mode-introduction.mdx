# 探索模式介绍

在 Grafana Alloy 中，**探索模式**是一个强大的工具，允许用户实时查询和可视化数据。无论你是初学者还是经验丰富的开发者，探索模式都能帮助你快速理解数据，发现潜在的问题或趋势。本文将详细介绍探索模式的功能、使用方法以及实际应用场景。

## 什么是探索模式？

探索模式是 Grafana Alloy 提供的一种交互式查询工具，允许用户在不保存查询结果的情况下，实时探索和分析数据。它特别适用于日志分析、指标监控和故障排查等场景。通过探索模式，你可以快速测试查询语句、调整可视化方式，并立即查看结果。

:::note
探索模式不会保存你的查询历史或结果，因此它非常适合临时性的数据探索和分析。
:::

## 探索模式的基本功能

探索模式提供了以下核心功能：

1. **实时查询**：输入查询语句后，立即查看结果。
2. **多种数据源支持**：支持 Prometheus、Loki、Elasticsearch 等多种数据源。
3. **灵活的可视化**：支持表格、折线图、柱状图等多种可视化方式。
4. **查询历史**：虽然不会保存查询结果，但可以查看最近的查询历史。

## 如何使用探索模式？

### 1. 进入探索模式

在 Grafana Alloy 的仪表板中，点击右上角的“探索”按钮，即可进入探索模式。

### 2. 选择数据源

在探索模式的顶部，选择一个数据源。例如，如果你要分析日志数据，可以选择 Loki 数据源。

### 3. 输入查询语句

在查询编辑器中输入查询语句。以下是一个简单的 Loki 查询示例：

```plaintext
{job="myapp"} |= "error"
```

该查询会返回所有包含“error”关键字的日志条目。

### 4. 查看结果

输入查询后，Grafana Alloy 会立即显示查询结果。你可以选择不同的可视化方式，例如表格或折线图。

### 5. 调整时间范围

在探索模式的右上角，你可以调整查询的时间范围。例如，选择“过去 1 小时”或自定义时间范围。

## 实际案例：日志分析

假设你正在监控一个 Web 应用程序，并希望分析过去 24 小时内出现的错误日志。以下是如何使用探索模式完成此任务的步骤：

1. 进入探索模式，选择 Loki 数据源。
2. 输入查询语句：`{job="webapp"} |= "error"`。
3. 调整时间范围为“过去 24 小时”。
4. 查看结果，并使用表格或折线图分析错误日志的趋势。

:::tip
如果你发现某个时间段的错误日志特别多，可以进一步细化查询，例如添加更多的过滤条件或使用正则表达式。
:::
