## 介绍

可访问性（Accessibility，简称 a11y）是指确保所有用户，包括残障人士，都能无障碍地使用你的应用程序。在 Vue.js 中，实现可访问性不仅是为了遵守法律和标准（如 [WCAG](https://www.w3.org/WAI/standards-guidelines/wcag/)），更是为了提升用户体验，让你的应用更具包容性。

本文将介绍如何在 Vue.js 中实现可访问性，涵盖从基础概念到实际应用的全面内容。

---

## 1. 理解可访问性的核心原则

可访问性遵循四个核心原则，简称 **POUR**：

1. **可感知性（Perceivable）**：用户必须能够感知到界面上的信息。
2. **可操作性（Operable）**：用户必须能够操作界面上的控件。
3. **可理解性（Understandable）**：用户必须能够理解界面上的信息和操作。
4. **健壮性（Robust）**：界面必须能够在不同的设备和辅助技术上正常工作。

在 Vue.js 中，我们可以通过以下方式实现这些原则。

---

## 2. 使用语义化 HTML

语义化 HTML 是实现可访问性的基础。Vue.js 允许我们使用标准的 HTML 元素，但有时开发者会忽略语义化的重要性。

### 示例：按钮 vs. div

```vue
<!-- 不推荐 -->
<div @click="handleClick">点击我</div>

<!-- 推荐 -->
<button @click="handleClick">点击我</button>
```

:::tip
使用 `<button>` 而不是 `<div>`，因为按钮元素具有内置的可访问性特性，例如键盘焦点和屏幕阅读器支持。
:::

---

## 3. 为交互元素添加 ARIA 属性

ARIA（Accessible Rich Internet Applications）是一组属性，用于增强 HTML 元素的可访问性。在 Vue.js 中，我们可以动态绑定 ARIA 属性。

### 示例：动态 ARIA 属性

```vue
<template>
  <button
    :aria-expanded="isExpanded"
    @click="toggleExpanded"
  >
    {{ isExpanded ? '收起' : '展开' }}
  </button>
</template>

<script>
export default {
  data() {
    return {
      isExpanded: false,
    };
  },
  methods: {
    toggleExpanded() {
      this.isExpanded = !this.isExpanded;
    },
  },
};
</script>
```

:::note
`aria-expanded` 属性用于指示一个元素是否展开，这对屏幕阅读器用户非常重要。
:::

---

## 4. 键盘导航支持

确保所有交互元素都可以通过键盘操作。例如，按钮、链接和表单控件应能够通过 `Tab` 键聚焦，并通过 `Enter` 或 `Space` 键触发。

### 示例：自定义组件的键盘支持

```vue
<template>
  <div
    tabindex="0"
    @keydown.enter="handleClick"
    @keydown.space="handleClick"
    @click="handleClick"
  >
    点击我
  </div>
</template>

<script>
export default {
  methods: {
    handleClick() {
      alert('点击成功！');
    },
  },
};
</script>
```

:::caution
如果使用 `div` 或 `span` 等非交互元素模拟按钮，务必手动添加键盘事件支持。
:::

## 5. 表单的可访问性

表单是用户与应用程序交互的重要部分。确保表单控件具有清晰的标签和错误提示。

### 示例：带标签和错误提示的输入框

```vue
<template>
  <form @submit.prevent="submitForm">
    <label for="username">用户名：</label>
    <input
      id="username"
      v-model="username"
      aria-describedby="username-error"
      required
    />
    <span id="username-error" v-if="error">{{ error }}</span>
    <button type="submit">提交</button>
  </form>
</template>

<script>
export default {
  data() {
    return {
      username: '',
      error: '',
    };
  },
  methods: {
    submitForm() {
      if (!this.username) {
        this.error = '用户名不能为空';
      } else {
        this.error = '';
        alert('提交成功！');
      }
    },
  },
};
</script>
```

:::tip
使用 `aria-describedby` 将错误提示与输入框关联，帮助屏幕阅读器用户理解错误信息。
:::

## 6. 实际案例：可访问的模态框

模态框是常见的 UI 组件，但如果实现不当，可能会对可访问性造成严重影响。

### 示例：可访问的模态框

```vue
<template>
  <div>
    <button @click="openModal">打开模态框</button>
    <div v-if="isOpen" role="dialog" aria-labelledby="modal-title">
      <h2 id="modal-title">这是一个模态框</h2>
      <p>模态框内容...</p>
      <button @click="closeModal">关闭</button>
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      isOpen: false,
    };
  },
  methods: {
    openModal() {
      this.isOpen = true;
    },
    closeModal() {
      this.isOpen = false;
    },
  },
};
</script>
```

:::warning
确保模态框在打开时将焦点移动到模态框内，并在关闭时将焦点返回到触发按钮。
:::

## 7. 总结

在 Vue.js 中实现可访问性并不复杂，但需要开发者在设计和开发过程中时刻关注。通过使用语义化 HTML、ARIA 属性、键盘导航支持和清晰的表单设计，你可以显著提升应用的可访问性。
