# Spring 请求参数

在Spring MVC中，请求参数是客户端发送给服务器的数据。这些数据可以通过URL、表单、HTTP头等多种方式传递。Spring MVC提供了多种方式来接收和处理这些请求参数，使得开发者能够轻松地与客户端进行数据交互。

## 1. 基本参数绑定

Spring MVC允许我们将请求参数直接绑定到控制器方法的参数上。这是最常见的处理请求参数的方式。

### 示例：基本参数绑定

```java
@RestController
public class MyController {

    @GetMapping("/greet")
    public String greet(@RequestParam String name) {
        return "Hello, " + name + "!";
    }
}
```

在这个示例中，`@RequestParam`注解用于将请求参数`name`绑定到方法参数`name`上。如果客户端发送请求`/greet?name=John`，服务器将返回`Hello, John!`。

:::note
如果请求参数名称与方法参数名称不一致，可以使用`@RequestParam("paramName")`来指定参数名称。
:::

## 2. 路径变量

路径变量是URL的一部分，通常用于标识资源。Spring MVC允许我们通过`@PathVariable`注解将路径变量绑定到方法参数上。

### 示例：路径变量

```java
@RestController
public class MyController {

    @GetMapping("/user/{id}")
    public String getUser(@PathVariable Long id) {
        return "User ID: " + id;
    }
}
```

在这个示例中，`@PathVariable`注解用于将路径变量`id`绑定到方法参数`id`上。如果客户端发送请求`/user/123`，服务器将返回`User ID: 123`。

## 3. 查询参数

查询参数是URL中`?`后面的部分，通常用于过滤或排序数据。Spring MVC允许我们通过`@RequestParam`注解将查询参数绑定到方法参数上。

### 示例：查询参数

```java
@RestController
public class MyController {

    @GetMapping("/search")
    public String search(@RequestParam String query, @RequestParam(required = false, defaultValue = "1") int page) {
        return "Searching for: " + query + " on page " + page;
    }
}
```

在这个示例中，`@RequestParam`注解用于将查询参数`query`和`page`绑定到方法参数上。如果客户端发送请求`/search?query=spring&page=2`，服务器将返回`Searching for: spring on page 2`。

:::tip
`required = false`表示该参数是可选的，`defaultValue`用于指定默认值。
:::

## 4. 表单数据

表单数据通常通过POST请求发送，Spring MVC允许我们通过`@ModelAttribute`注解将表单数据绑定到对象上。

### 示例：表单数据

```java
@RestController
public class MyController {

    @PostMapping("/register")
    public String register(@ModelAttribute User user) {
        return "Registered user: " + user.getName() + ", Email: " + user.getEmail();
    }
}

public class User {
    private String name;
    private String email;

    // Getters and Setters
}
```

在这个示例中，`@ModelAttribute`注解用于将表单数据绑定到`User`对象上。如果客户端发送POST请求并提交表单数据`name=John&email=john@example.com`，服务器将返回`Registered user: John, Email: john@example.com`。

## 5. 实际案例

假设我们正在开发一个简单的博客系统，用户可以创建和查看博客文章。我们可以使用Spring MVC来处理用户的请求参数。

### 示例：博客系统

```java
@RestController
public class BlogController {

    @GetMapping("/posts/{id}")
    public String getPost(@PathVariable Long id) {
        return "Fetching post with ID: " + id;
    }

    @PostMapping("/posts")
    public String createPost(@RequestParam String title, @RequestParam String content) {
        return "Created post with title:  + title +  and content: " + content;
    }
}
```

在这个示例中，`getPost`方法用于获取特定ID的博客文章，`createPost`方法用于创建新的博客文章。

## 6. 总结

Spring MVC提供了多种方式来处理请求参数，包括基本参数绑定、路径变量、查询参数和表单数据。通过合理使用这些功能，我们可以轻松地与客户端进行数据交互，构建功能强大的Web应用程序。
