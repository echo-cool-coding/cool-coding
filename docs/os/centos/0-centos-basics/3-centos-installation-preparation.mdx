# CentOS 安装准备

在开始安装CentOS之前，进行充分的准备工作是非常重要的。这不仅可以帮助你避免在安装过程中遇到问题，还能确保系统安装后能够正常运行。本文将逐步讲解安装CentOS前的准备工作，并提供实际案例帮助你更好地理解。

## 1. 硬件要求

在安装CentOS之前，首先需要确保你的计算机满足最低硬件要求。以下是CentOS 7和CentOS 8的最低硬件要求：

- **处理器（CPU）**：至少1 GHz的处理器。
- **内存（RAM）**：至少1 GB（建议2 GB或更多）。
- **硬盘空间**：至少20 GB的可用空间。
- **显卡**：支持1024x768分辨率的显卡。
- **网络**：网络连接（可选，但建议）。

:::note
如果你的计算机不满足这些要求，安装过程可能会失败，或者系统运行时会非常缓慢。
:::

## 2. 下载CentOS镜像

在确认硬件满足要求后，下一步是下载CentOS的安装镜像。你可以从CentOS的官方网站下载最新的镜像文件。

1. 访问 [CentOS官方网站](https://www.centos.org/download/)。
2. 选择适合你需求的版本（如CentOS 7或CentOS 8）。
3. 下载ISO镜像文件。

:::tip
建议下载`DVD ISO`镜像，因为它包含了大多数常用的软件包，安装时无需联网。
:::

## 3. 创建启动盘

下载完ISO镜像后，你需要将其写入U盘或DVD，以便从该介质启动并安装CentOS。

### 使用U盘创建启动盘

1. 插入U盘（至少8GB）。
2. 使用工具如`Rufus`（Windows）或`dd`命令（Linux/macOS）将ISO镜像写入U盘。

#### 使用`dd`命令创建启动盘（Linux/macOS）

```bash
sudo dd if=/path/to/centos.iso of=/dev/sdX bs=4M status=progress
```

- `if`：指定ISO文件的路径。
- `of`：指定U盘的设备路径（如`/dev/sdX`，请确保选择正确的设备）。
- `bs`：设置块大小。
- `status=progress`：显示进度。

:::caution
请务必确认`of`参数指定的设备路径正确，否则可能会覆盖其他磁盘数据。
:::

### 使用DVD创建启动盘

如果你选择使用DVD，可以使用光盘刻录软件（如`ImgBurn`或`Brasero`）将ISO镜像刻录到DVD中。

## 4. 设置BIOS/UEFI启动顺序

在启动计算机之前，你需要确保计算机从U盘或DVD启动。这通常需要在BIOS/UEFI设置中调整启动顺序。

1. 重启计算机并进入BIOS/UEFI设置界面（通常按`F2`、`F12`、`DEL`或`ESC`键）。
2. 找到“启动顺序”或“Boot Order”选项。
3. 将U盘或DVD设置为第一启动项。
4. 保存设置并退出。

:::warning
不同计算机的BIOS/UEFI界面可能有所不同，请参考你的计算机手册。
:::

## 5. 分区规划

在安装CentOS之前，建议提前规划好磁盘分区。常见的分区方案包括：

- **/boot**：用于存放启动文件，建议分配500MB。
- **/（根分区）**：用于存放系统文件和应用程序，建议分配20GB或更多。
- **/home**：用于存放用户数据，建议分配剩余空间。
- **swap**：用于虚拟内存，建议分配与内存大小相同或两倍。

:::note
如果你不确定如何分区，可以选择自动分区，安装程序会为你创建默认的分区方案。
:::

## 6. 网络配置

如果你计划在安装过程中联网下载软件包或更新系统，建议提前配置好网络。

1. 确保计算机连接到网络（有线或无线）。
2. 在安装过程中，选择“网络和主机名”选项，配置网络设置。

## 7. 实际案例

假设你有一台旧笔记本电脑，配置为2GB内存、500GB硬盘，你计划在这台电脑上安装CentOS 7用于学习Linux。

1. 检查硬件配置，确认满足最低要求。
2. 下载CentOS 7的`DVD ISO`镜像。
3. 使用`dd`命令将ISO镜像写入U盘。
4. 进入BIOS设置，将U盘设置为第一启动项。
5. 启动计算机，进入CentOS安装界面。
6. 选择手动分区，分配500MB给`/boot`，20GB给`/`，剩余空间给`/home`，并创建2GB的`swap`分区。
7. 配置网络，确保可以联网下载软件包。
8. 完成安装并重启系统。

## 总结

在安装CentOS之前，进行充分的准备工作是确保安装过程顺利的关键。本文详细介绍了硬件要求、下载镜像、创建启动盘、设置BIOS/UEFI、分区规划和网络配置等步骤。通过实际案例，你可以更好地理解这些步骤在实际中的应用。
