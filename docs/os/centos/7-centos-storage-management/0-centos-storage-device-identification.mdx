# CentOS 存储设备识别

在CentOS中，存储设备的管理是系统管理员和开发人员必须掌握的基本技能之一。无论是硬盘、SSD、USB驱动器还是其他存储设备，了解如何识别和管理这些设备对于系统的正常运行至关重要。本文将逐步介绍如何在CentOS中识别存储设备，并提供实际案例和代码示例。

## 1. 存储设备的基本概念

在Linux系统中，存储设备通常以文件的形式存在于`/dev`目录下。每个设备都有一个唯一的标识符，例如`/dev/sda`、`/dev/sdb`等。这些设备可以是物理硬盘、分区、虚拟磁盘等。

:::note
- `/dev/sda`：通常表示系统中的第一块硬盘。
- `/dev/sdb`：通常表示系统中的第二块硬盘。
- `/dev/sda1`：表示第一块硬盘上的第一个分区。
:::

## 2. 使用`lsblk`命令识别存储设备

`lsblk`命令是Linux中用于列出块设备（如硬盘、分区等）的常用工具。它提供了一个清晰的树状结构，显示设备及其分区的关系。

```bash
lsblk
```

**输出示例：**

```
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0   100G  0 disk 
├─sda1   8:1    0    50G  0 part /
├─sda2   8:2    0    30G  0 part /home
└─sda3   8:3    0    20G  0 part [SWAP]
sdb      8:16   0   200G  0 disk 
└─sdb1   8:17   0   200G  0 part /data
```

在这个示例中，`sda`和`sdb`是两块硬盘，`sda`有三个分区，而`sdb`只有一个分区。

## 3. 使用`fdisk`查看设备详细信息

`fdisk`是一个功能强大的工具，用于查看和管理磁盘分区。通过`fdisk -l`命令，你可以查看系统中所有磁盘的详细信息。

```bash
sudo fdisk -l
```

**输出示例：**

```
Disk /dev/sda: 100 GiB, 107374182400 bytes, 209715200 sectors
Disk model: VIRTUAL DISK    
Units: sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disklabel type: dos
Disk identifier: 0x12345678

Device     Boot   Start       End   Sectors  Size Id Type
/dev/sda1  *       2048 104857599 104855552  50G 83 Linux
/dev/sda2      104857600 167772159  62914560  30G 83 Linux
/dev/sda3      167772160 209715199  41943040  20G 82 Linux swap / Solaris
```

在这个示例中，`fdisk -l`显示了`/dev/sda`的详细信息，包括分区表、分区大小和类型。

## 4. 使用`blkid`查看设备UUID和文件系统类型

`blkid`命令用于显示块设备的UUID和文件系统类型。UUID是设备的唯一标识符，通常用于在`/etc/fstab`中挂载设备。

```bash
sudo blkid
```

**输出示例：**

```
/dev/sda1: UUID="123e4567-e89b-12d3-a456-426614174000" TYPE="ext4" PARTUUID="12345678-01"
/dev/sda2: UUID="987e6543-e89b-12d3-a456-426614174000" TYPE="ext4" PARTUUID="12345678-02"
/dev/sda3: UUID="456e1234-e89b-12d3-a456-426614174000" TYPE="swap" PARTUUID="12345678-03"
/dev/sdb1: UUID="789e1234-e89b-12d3-a456-426614174000" TYPE="xfs" PARTUUID="12345678-04"
```

在这个示例中，`blkid`显示了每个分区的UUID和文件系统类型。

## 5. 实际案例：识别新插入的USB设备

假设你插入了一个新的USB驱动器，你需要识别它并将其挂载到系统中。

1. 首先，使用`lsblk`查看新设备：

```bash
lsblk
```

**输出示例：**

```
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0   100G  0 disk 
├─sda1   8:1    0    50G  0 part /
├─sda2   8:2    0    30G  0 part /home
└─sda3   8:3    0    20G  0 part [SWAP]
sdb      8:16   0   200G  0 disk 
└─sdb1   8:17   0   200G  0 part /data
sdc      8:32   1    15G  0 disk 
└─sdc1   8:33   1    15G  0 part 
```

在这个示例中，`sdc`是新插入的USB设备。

2. 使用`blkid`查看USB设备的UUID和文件系统类型：

```bash
sudo blkid /dev/sdc1
```

**输出示例：**

```
/dev/sdc1: UUID="abc12345-e89b-12d3-a456-426614174000" TYPE="vfat"
```

3. 创建一个挂载点并挂载USB设备：

```bash
sudo mkdir /mnt/usb
sudo mount /dev/sdc1 /mnt/usb
```

4. 使用`df -h`查看挂载情况：

```bash
df -h
```

**输出示例：**

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   20G   30G  40% /
/dev/sda2        30G   10G   20G  33% /home
/dev/sdb1       200G   50G  150G  25% /data
/dev/sdc1        15G    1G   14G   7% /mnt/usb
```

## 6. 总结

在CentOS中识别存储设备是系统管理的基础技能。通过使用`lsblk`、`fdisk`和`blkid`等工具，你可以轻松地查看和管理系统中的存储设备。本文还提供了一个实际案例，展示了如何识别和挂载新插入的USB设备。
