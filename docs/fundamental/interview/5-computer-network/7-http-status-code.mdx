## 介绍

HTTP状态码（HTTP Status Code）是服务器在响应客户端请求时返回的三位数字代码。它们用于表示请求的处理结果，帮助开发者和用户了解请求是否成功、是否需要重定向、是否存在错误等。HTTP状态码是Web开发中不可或缺的一部分，掌握它们对于调试和优化应用程序至关重要。

HTTP状态码分为五类，每类以不同的数字开头：

1. **1xx（信息性状态码）**：表示请求已被接收，继续处理。
2. **2xx（成功状态码）**：表示请求已成功被服务器接收、理解并处理。
3. **3xx（重定向状态码）**：表示需要进一步操作以完成请求。
4. **4xx（客户端错误状态码）**：表示客户端发送的请求有误。
5. **5xx（服务器错误状态码）**：表示服务器在处理请求时发生了错误。

接下来，我们将逐一介绍这些状态码，并通过实际案例帮助你更好地理解它们的应用场景。

---

## 1xx 信息性状态码

1xx状态码表示请求已被接收，服务器正在处理。这类状态码通常用于临时响应，客户端需要继续等待服务器的最终响应。

### 常见状态码
- **100 Continue**：客户端应继续发送请求的剩余部分。通常用于POST请求中，客户端发送请求头后，服务器确认可以继续发送请求体。
- **101 Switching Protocols**：服务器同意客户端请求，切换协议（例如从HTTP切换到WebSocket）。

:::note
1xx状态码在实际开发中较少直接使用，通常由浏览器或HTTP库自动处理。
:::

---

## 2xx 成功状态码

2xx状态码表示请求已成功被服务器接收、理解并处理。这是开发者最希望看到的状态码。

### 常见状态码
- **200 OK**：请求成功。这是最常见的状态码，表示服务器已成功处理请求并返回了所需的数据。
- **201 Created**：请求成功，并且服务器创建了新的资源（例如在POST请求中创建新用户）。
- **204 No Content**：请求成功，但响应中没有返回任何内容（例如在DELETE请求中删除资源后）。

#### 示例
假设你向服务器发送一个GET请求以获取用户信息：

```http
GET /users/1 HTTP/1.1
Host: api.example.com
```

服务器成功处理请求并返回用户信息：

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com"
}
```

## 3xx 重定向状态码

3xx状态码表示需要进一步操作以完成请求。通常用于URL重定向或资源迁移。

### 常见状态码
- **301 Moved Permanently**：请求的资源已永久移动到新位置，客户端应使用新的URL。
- **302 Found**：请求的资源临时移动到新位置，客户端应使用新的URL。
- **304 Not Modified**：资源未修改，客户端可以使用缓存的版本。

#### 示例
假设你访问一个旧URL，服务器返回301状态码并重定向到新URL：

```http
GET /old-page HTTP/1.1
Host: example.com
```

服务器响应：

```http
HTTP/1.1 301 Moved Permanently
Location: https://example.com/new-page
```

---

## 4xx 客户端错误状态码

4xx状态码表示客户端发送的请求有误，服务器无法处理。

### 常见状态码
- **400 Bad Request**：请求无效，服务器无法理解（例如请求体格式错误）。
- **401 Unauthorized**：请求需要身份验证，客户端未提供有效的凭据。
- **403 Forbidden**：服务器理解请求，但拒绝执行（例如权限不足）。
- **404 Not Found**：请求的资源不存在。

#### 示例
假设你尝试访问一个不存在的资源：

```http
GET /non-existent-page HTTP/1.1
Host: example.com
```

服务器响应：

```http
HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "Resource not found"
}
```

---

## 5xx 服务器错误状态码

5xx状态码表示服务器在处理请求时发生了错误。

### 常见状态码
- **500 Internal Server Error**：服务器遇到意外错误，无法完成请求。
- **502 Bad Gateway**：服务器作为网关或代理时，从上游服务器收到无效响应。
- **503 Service Unavailable**：服务器暂时无法处理请求（例如服务器过载或维护中）。

#### 示例
假设服务器在处理请求时发生内部错误：

```http
GET /users HTTP/1.1
Host: api.example.com
```

服务器响应：

```http
HTTP/1.1 500 Internal Server Error
Content-Type: application/json

{
  "error": "Internal server error"
}
```

## 实际应用场景

### 场景1：表单提交
当用户提交表单时，服务器可能返回以下状态码：
- **200 OK**：表单提交成功。
- **400 Bad Request**：表单数据格式错误。
- **401 Unauthorized**：用户未登录。

### 场景2：API调用
当调用API时，服务器可能返回以下状态码：
- **200 OK**：API调用成功并返回数据。
- **404 Not Found**：请求的资源不存在。
- **500 Internal Server Error**：服务器内部错误。

## 总结

HTTP状态码是Web开发中非常重要的工具，它们帮助开发者和用户了解请求的处理结果。通过掌握1xx到5xx状态码的含义和应用场景，你可以更好地调试和优化应用程序。
