# Git 安装配置

Git 是一个分布式版本控制系统，广泛用于跟踪代码的变化并协作开发项目。在开始使用 Git 之前，首先需要将其安装到你的计算机上，并进行一些基本的配置。本教程将逐步指导你完成 Git 的安装和配置过程。

## 1. 安装 Git

### 在 Windows 上安装 Git

1. **下载 Git 安装程序**  
   访问 [Git 官方网站](https://git-scm.com/) 并下载适用于 Windows 的安装程序。

2. **运行安装程序**  
   双击下载的安装程序，按照提示完成安装。在安装过程中，你可以选择默认选项，但请注意以下几点：
   - 选择 Git 的默认编辑器（推荐使用 Vim 或 Nano）。
   - 选择“Git from the command line and also from 3rd-party software”选项，以便在命令行中使用 Git。

3. **验证安装**  
   安装完成后，打开命令提示符（cmd）并输入以下命令：
   ```bash
   git --version
   ```
   如果安装成功，你将看到 Git 的版本号，例如：
   ```
   git version 2.34.1
   ```

### 在 macOS 上安装 Git

1. **使用 Homebrew 安装 Git**  
   如果你已经安装了 Homebrew，可以通过以下命令安装 Git：
   ```bash
   brew install git
   ```

2. **验证安装**  
   安装完成后，打开终端并输入以下命令：
   ```bash
   git --version
   ```
   如果安装成功，你将看到 Git 的版本号。

### 在 Linux 上安装 Git

1. **使用包管理器安装 Git**  
   在大多数 Linux 发行版上，你可以使用包管理器来安装 Git。例如，在 Ubuntu 上，可以使用以下命令：
   ```bash
   sudo apt-get update
   sudo apt-get install git
   ```

2. **验证安装**  
   安装完成后，打开终端并输入以下命令：
   ```bash
   git --version
   ```
   如果安装成功，你将看到 Git 的版本号。

## 2. 配置 Git

安装完成后，你需要对 Git 进行一些基本配置，以便在使用时能够正确识别你的身份。

### 设置用户名和邮箱

Git 使用用户名和邮箱来标识提交记录的作者。你可以通过以下命令设置全局的用户名和邮箱：

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

:::tip
你可以通过以下命令查看当前的 Git 配置：
```bash
git config --list
```
:::

### 配置默认文本编辑器

Git 默认使用 Vim 作为文本编辑器，但你可以将其更改为你喜欢的编辑器。例如，如果你想使用 Nano 作为默认编辑器，可以运行以下命令：

```bash
git config --global core.editor "nano"
```

### 配置行尾符处理

在不同的操作系统中，行尾符的处理方式不同。为了避免在跨平台协作时出现问题，你可以配置 Git 自动处理行尾符：

```bash
git config --global core.autocrlf true  # 适用于 Windows
git config --global core.autocrlf input # 适用于 macOS 和 Linux
```

## 3. 验证 Git 配置

完成配置后，你可以通过以下命令验证 Git 的配置是否正确：

```bash
git config --list
```

这将列出所有的 Git 配置项，确保你的用户名、邮箱和编辑器等设置都已正确配置。

## 4. 实际案例

假设你正在与团队协作开发一个项目，并且你刚刚在一台新电脑上安装了 Git。你需要确保 Git 能够正确识别你的身份，以便在提交代码时能够正确记录作者信息。

1. **设置用户名和邮箱**  
   你运行以下命令来设置你的用户名和邮箱：
   ```bash
   git config --global user.name "John Doe"
   git config --global user.email "john.doe@example.com"
   ```

2. **验证配置**  
   你运行以下命令来验证配置是否正确：
   ```bash
   git config --list
   ```
   输出如下：
   ```
   user.name=John Doe
   user.email=john.doe@example.com
   core.editor=nano
   core.autocrlf=true
   ```

3. **提交代码**  
   现在，当你提交代码时，Git 将使用你配置的用户名和邮箱来标识提交记录。

## 5. 总结

在本教程中，我们学习了如何在不同操作系统上安装 Git，并进行了基本的配置。我们还通过实际案例展示了如何验证配置是否正确。通过这些步骤，你现在已经准备好开始使用 Git 来管理你的代码了。
